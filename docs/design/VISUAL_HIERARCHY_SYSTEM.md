# VISUAL_HIERARCHY_SYSTEM.md
## SendMeStudio — Editorial Visual Hierarchy & Cognitive Focus System
### v1.0 — Luxury Interface Composition Rules

---

# PURPOSE

This document defines the visual hierarchy, focus orchestration, cognitive load control, editorial composition, spacing rhythm, ambient intelligence behavior, and interface narrative philosophy for SendMeStudio.

The goal is NOT to create a dashboard.

The goal is to create:

- an editorial operating system
- a luxury conversational canvas
- an ambient intelligence interface
- a cinematic beauty experience

---

# CORE PRINCIPLE

Most dashboards fail because:

- everything competes
- every panel screams
- every card has equal weight
- the eye has no destination

Luxury interfaces do the opposite.

Luxury interfaces:

- guide the eye
- reduce cognitive noise
- create visual silence
- establish one dominant focus
- lower secondary contrast
- create emotional breathing space

---

# THE GOLDEN RULE

# ONLY ONE PRIMARY FOCUS PER VIEW

```yaml
hero_focus:
  max_primary_focus_per_screen: 1
```

Every screen must have:

- 1 dominant element
- 2 secondary support zones
- tertiary ambient information

Never multiple competing hero blocks.

# THE "NO BOX EVERYTHING" RULE

This is CRITICAL.

Most premium interfaces do NOT wrap every element in cards.

Luxury interfaces use:

- alignment
- spacing
- typography
- rhythm
- composition

to create grouping.

NOT borders.

---

# RULE

```yaml
boxing_rules:
  only_box_when:
    - interaction_required
    - hierarchy_needed
    - focus_needed

  avoid_boxing:
    - metadata
    - labels
    - passive_information
    - secondary_actions
```
